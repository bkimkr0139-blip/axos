# 03 · AX전환 플레이북 — "물 흐르듯, 사내 저항 없이"

> AX전환의 실패는 기술이 아니라 **사람**에서 온다. "AI가 내 일을 빼앗는다"는 두려움, "또 새 시스템이냐"는 피로,
> "내 데이터를 왜 주냐"는 불신. 이 플레이북은 그 저항을 **설계로** 흡수해, 전환이 강요가 아니라 흐름이 되게 한다.

핵심 4원칙:
1. **빼앗지 말고 덜어준다** — AI는 사람을 대체가 아니라 잡일에서 해방. 첫 메시지가 ROI보다 중요.
2. **읽기 먼저, 쓰기 나중** — 처음엔 아무것도 바꾸지 않는다(읽기 전용). 신뢰가 쓰기를 연다.
3. **작게, 자주, 보이게** — 빅뱅 금지. 2주 단위로 작은 승리를 가시화.
4. **사람이 항상 핸들을 쥔다** — HITL 기본. 자동화 레벨은 현장이 올린다, 위에서 내리꽂지 않는다.

---

## 1. 자동화 성숙도 모델 (L0~L4) — 모든 시나리오가 이 사다리를 오른다

| 레벨 | 이름 | AI 역할 | 사람 역할 | 신뢰 조건 |
|------|------|---------|-----------|-----------|
| **L0** | 관찰 | 데이터만 보여줌 | 모든 판단·실행 | — |
| **L1** | 제안 | 인사이트·추천 제시 | 판단·실행 | 추천 정확도 체감 |
| **L2** | 초안 | 행동 초안 작성(메일/발주서 draft) | 검토·승인·실행 | 초안 채택률↑ |
| **L3** | 승인형 실행 | 실행까지 하되 사람 승인 게이트 | 승인/반려 | 승인 후 결과 신뢰 |
| **L4** | 조건부 자동 | 한도·시나리오 내 자동 실행 | 예외·감사 | 오류율<임계, 감사 통과 |

> **핵심: 레벨은 시나리오마다 따로 오른다.** 발주는 L3인데 메일초안은 L2, 문서검색은 L4일 수 있다.
> 한 시나리오가 L4여도 새 시나리오는 항상 L0/L1부터. **자동화 레벨 상향은 현장 합의로만.**

이 사다리는 docs/04의 브리지 `approval_policy`와 docs/02의 통합 성숙도(I0~I4)와 직접 연결된다.

## 2. 단계별 전환 로드맵 (90일 → 12개월)

### Phase 0 — 토대 (0~2주) · "아무것도 바꾸지 않는다"
- 읽기 전용 연결(Mail/File → 문서 기억). 기존 업무 그대로.
- 챔피언(현업 1~2명) 선정 — 강요 대상이 아니라 **공동 설계자**로.
- 메시지: "여러분 일을 바꾸려는 게 아니라, 흩어진 정보를 AI가 대신 찾아주게 하려는 것."

### Phase 1 — 체감 (2~6주) · "작은 해방감"
- AI Copilot(STEP5) L1: "우리 회사 문서에 물어보면 답이 나온다".
- 첫 승리 1개: 매일 30분 걸리던 자료 취합을 3분으로. → **시간 절감을 숫자로 게시**.
- 저항 흡수: 잘못된 답엔 "출처 보기"로 검증 가능 → 불신을 투명성으로.

### Phase 2 — 신뢰 (6~12주) · "초안을 써준다"
- L2: 회의록 요약·메일 초안·보고서 초안 자동 작성, 사람이 마무리.
- 도메인 Agent 1종 수직슬라이스(권장: SCM Agent 재고→발주추천, L1→L2).
- 저항 흡수: "초안일 뿐, 보내는 건 당신" → 통제감 유지.

### Phase 3 — 실행 (3~6개월) · "분석형 → 실행형 전환점"
- L3: 승인형 실행. 발주 추천→승인→ERP 반영(docs/02 I3). **여기서 AXOS가 진짜 "실행형"이 된다.**
- 운영 대시보드(STEP10)로 ROI·절감시간 가시화 → 경영진 신뢰·확산 동력.
- 6개 Agent 수평 확장.

### Phase 4 — 정착·고도화 (6~12개월) · "AI 직원"
- L4: 검증된 시나리오 한정 조건부 자동(한도 내 자동 발주, 임계 자동 대응).
- 예측 AI(STEP8) 상시화, Agent Memory(STEP9)로 판단 품질 자가 향상.
- 거버넌스 정례화(분기 감사·레벨 재평가).

## 3. 저항 유형별 대응 (Resistance Playbook)

| 저항 | 누가 | 진짜 원인 | 대응 설계 |
|------|------|-----------|-----------|
| "내 일을 빼앗긴다" | 현업 실무 | 고용 불안 | 잡일 제거로 포지셔닝. 절감시간을 "더 가치있는 일"로 재배치 명시. 챔피언이 동료에 전파 |
| "또 새 시스템이냐" | 현업 전반 | 도입 피로 | 새 화면 최소화 — 기존 메신저/메일/Base44 안에서 동작. 별도 로그인·학습 곡선 제거 |
| "AI를 못 믿겠다" | 실무·관리자 | 블랙박스 | 모든 답에 근거·출처(Decision Envelope의 evidence). HITL 게이트로 사람이 최종 결정 |
| "내 데이터를 왜 주냐" | 부서장 | 통제권 상실 | UC 권한이 부서 통제 유지. 읽기 전용 시작. 마스킹(HR). 어디에 쓰였는지 감사 가능 |
| "ERP 건드리지 마라" | IT/시스템 | 장애 책임 | 읽기 전용 계정 시작. 쓰기는 전용계정·한도·드라이런·보상. IT를 통합 공동소유자로 |
| "성과가 안 보인다" | 경영진 | ROI 불확실 | Phase마다 측정 가능한 1승리. 대시보드 ROI/절감시간 상시 노출 |

## 4. 변화관리 운영 (Operating Rhythm)

- **챔피언 네트워크**: 부서별 1명. AXOS 팀과 격주 미팅. 현장 요구가 백로그가 됨(하향식 금지).
- **2주 스프린트 + 데모**: 매 스프린트 끝에 "이번에 덜어준 잡일" 데모. 작은 승리의 의식화.
- **투명 게시판**: 절감시간·자동처리건수·정확도를 사내에 공개 → 신뢰는 숫자로 쌓인다.
- **레벨업 회의**: 시나리오의 L상향은 현장+오너+거버넌스 3자 합의로만. 강제 승급 금지.
- **킬 스위치**: 어떤 자동화든 현장이 즉시 끌 수 있는 버튼. "언제든 멈출 수 있다"는 안전감이 수용을 높인다.

## 5. 측정 지표 (전환이 "흐르고 있는지" 보는 계기판)

| 범주 | 지표 | 의미 |
|------|------|------|
| 채택 | 주간 활성 사용자/부서, 질의수 | 자발적 사용 = 수용 |
| 신뢰 | 추천 채택률, 승인 통과율, 반려율 | 판단 품질·신뢰도 |
| 가치 | 절감시간, 자동처리 건수/성공률, ROI | 경영 정당화 |
| 안전 | 오류율, 롤백/보상 발생, 감사 예외 | 통제 건전성 |
| 성숙 | 시나리오별 L레벨 분포 | 전환 진척 |

이 지표들은 STEP10 운영 대시보드의 데이터 모델과 동일하다(→ docs/01 STEP10, docs/06).

## 6. 안티패턴 (하지 말 것)

- ❌ 빅뱅 전사 동시 적용 — 저항을 한꺼번에 산다.
- ❌ 첫 단계부터 쓰기/자동화 — 한 번의 잘못된 자동 발주가 신뢰를 영구 파괴.
- ❌ ROI를 첫 메시지로 — 현업은 "내 일이 어떻게 되나"가 먼저. ROI는 경영진 언어.
- ❌ 새 시스템 학습 강요 — 기존 도구 안으로 들어가라.
- ❌ 블랙박스 — 근거 없는 답은 신뢰를 못 얻는다.
- ❌ 현장 합의 없는 레벨 상향 — 통제감 상실 = 저항 재점화.
