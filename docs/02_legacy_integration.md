# 02 · 레거시 연계·통합 전략 (ERP/MES/CRM/SCM/HR/Mail/File/DB/IoT)

> 핵심 원칙: **레거시를 걷어내지 않는다(Strangler, not Rip-and-Replace).**
> 기존 ERP/MES를 그대로 두고, AXOS는 그 위에 **읽기(이해)** 와 **쓰기(실행)** 를 감싸는 어댑터 계층으로 붙는다.
> 그래야 "분석형"이 아니라 "실행형"이 되면서도, 기존 시스템·기존 담당자의 저항을 만들지 않는다.

---

## 1. 통합의 두 방향 — 읽기와 쓰기를 분리한다

AXOS와 레거시의 접점은 **반드시 두 방향으로 분리**한다. 이 분리가 안전성과 수용성의 핵심이다.

| 방향 | 무엇 | 레이어 | 위험도 | 원칙 |
|------|------|--------|--------|------|
| **읽기 (Inbound)** | 레거시 데이터를 Lakehouse로 수집 | 판단(Databricks) | 낮음 | 광범위·적극적으로. 읽기는 레거시를 바꾸지 않는다 |
| **쓰기 (Outbound)** | AXOS 결정을 레거시에 반영(발주/전표/상태변경) | 실행(n8n) | 높음 | 좁게·승인 게이트·멱등성·보상 트랜잭션 |

> 처음에는 **읽기만** 연결한다("AI가 우리 데이터를 이해한다"). 쓰기는 신뢰가 쌓인 뒤 시나리오별로 하나씩 연다. → docs/03 자동화 성숙도.

## 2. 소스별 연계 패턴 (9종)

### 2.1 ERP (예: SAP, 더존, 영림원, Oracle EBS)
- **읽기**: CDC 우선(DB 트랜잭션 로그/타임스탬프 델타). 불가 시 야간 배치(JDBC) + OData/REST.
  - 대상: 수주/발주/재고/회계전표/마스터(품목·거래처).
  - 적재: Bronze(원본) → Silver(정규화/조인) → Gold(KPI/예측 피처).
- **쓰기**: ERP 표준 API/BAPI/IDoc 또는 RPA 최후수단. **n8n 어댑터 워크플로우**가 담당, 브리지 승인 통과분만.
  - 멱등키: `axos_decision_id` 를 ERP 참조필드에 기록 → 중복 발주 방지.
- **저항 포인트**: "ERP 건드리지 마라". → 1단계는 읽기 전용 계정. 쓰기는 별도 통합사용자+한도+로그로 명확히.

### 2.2 MES (제조 실행)
- **읽기**: 설비/공정 이벤트 스트리밍(OPC-UA/Kafka/MQTT) → Structured Streaming → Bronze.
- **용도**: 불량예측(STEP8)·Quality Agent·설비고장예측 입력.
- **쓰기**: 보통 읽기 전용 권장(현장 안전). 알람/작업지시는 사람 경유.

### 2.3 CRM (Salesforce/HubSpot/자체)
- **읽기**: REST 배치(고객/상담/파이프라인). Sales Agent 입력.
- **쓰기**: 활동 로그/태스크 생성 정도부터(저위험). 단계 변경은 승인.

### 2.4 SCM
- **읽기**: 재고/리드타임/공급사 배치+CDC. SCM Agent·납기예측 핵심 입력.
- **쓰기**: 발주는 ERP 경유(2.1). 안전재고 파라미터 제안은 읽기+추천.

### 2.5 HR
- **읽기**: 인사/근태/조직 배치. **민감정보** → UC 컬럼 마스킹·접근통제 필수(docs/06).
- **쓰기**: 원칙적으로 안 함. 인사이트·리포트만.

### 2.6 Mail
- **읽기**: 메일 API(Graph/Gmail) 본문·첨부 → Bronze + Vector(문서 기억).
- **활용**: 회의록/계약/업무지시 자동 인덱싱. 기존 n8n `01 document-ingest` 재사용.
- **쓰기**: 메일 발송 = 기존 `이메일 발송 도구` 워크플로우. 외부 발송은 승인.

### 2.7 File
- **읽기**: 파일서버/SharePoint/오브젝트스토리지 → UC Volume → Vector.
- **활용**: 문서 기억(STEP9). 변경 감지 시 `09 vector-reindex`.

### 2.8 Database (레거시 DB 일반)
- **읽기**: JDBC/CDC. 레거시 자체 DB(그룹웨어/자체개발)를 Lakehouse로.
- **쓰기**: 지양. 필요 시 전용 스테이징 테이블 경유.

### 2.9 IoT
- **읽기**: 센서 스트리밍(MQTT/Kafka) → DLT → Bronze. 설비고장예측·임계 알림.
- **쓰기**: 제어계통 직접 쓰기 금지. 임계 초과 → `08 notify` 알림까지만.

## 3. 표준 연계 아키텍처

```
                   ┌──────────── 읽기(Inbound) ────────────┐
 ERP/CRM/SCM/HR ──▶ CDC/배치(JDBC,REST,OData) ─┐
 MES/IoT        ──▶ 스트리밍(OPC-UA/MQTT/Kafka)─┼─▶ Bronze ─▶ Silver ─▶ Gold (Delta/UC)
 Mail/File      ──▶ API/볼륨 동기화 ────────────┘                          │
                                                                          ▼
                                                              판단(Databricks) — 예측/Agent
                                                                          │ Decision Envelope
                   ┌──────────── 쓰기(Outbound) ───────────┐              ▼
 ERP 발주/전표  ◀── n8n 어댑터(승인·멱등·보상) ◀── 실행 브리지 ◀──────────┘
 메일/메신저    ◀── 기존 n8n 워크플로우
 이슈/태스크    ◀── 도메인 시스템 API
```

- **어댑터 레지스트리**: 시스템별 어댑터를 계약(`contracts/agents`·`contracts/bridge`)으로 표준화. 레거시 1종 추가 = 어댑터 1개 추가.
- **안티-커럽션 레이어(ACL)**: 레거시의 지저분한 스키마를 AXOS 표준 모델로 번역하는 Silver 변환에서 격리.

## 4. 통합 성숙도 (시스템별로 독립 진행)

| 레벨 | 의미 | ERP 예시 |
|------|------|----------|
| **I0** | 미연결 | — |
| **I1** | 읽기(배치) | 야간 재고 스냅샷 적재 |
| **I2** | 읽기(준실시간/CDC) | 발주/재고 변경 실시간 반영 |
| **I3** | 쓰기(승인 필수) | AI 발주 추천 → 사람 승인 → ERP 발주 |
| **I4** | 쓰기(조건부 자동) | 신뢰 시나리오 한정 자동 발주(한도 내) |

> 같은 회사 안에서도 ERP는 I3, HR은 I1, Mail은 I2처럼 **시스템마다 다른 속도**로 올라간다. 강요하지 않는다.

## 5. 안전장치 (쓰기 통합 필수 요건)

1. **멱등성**: 모든 쓰기에 `axos_decision_id` 멱등키. 재시도해도 중복 실행 없음.
2. **보상 트랜잭션**: 실패 시 롤백/취소 경로 정의(예: 발주 실패→취소요청, 알림 발송).
3. **한도(Guardrail)**: 금액/수량/빈도 한도. 초과 시 자동 승인 게이트 강제.
4. **드라이런**: 모든 어댑터는 `dryRun` 지원 → 실제 쓰기 전 시뮬레이션·시연.
5. **감사**: 누가(Agent/사람)·언제·무엇을·왜(근거 봉투) 전부 기록 → 대시보드/감사.
6. **전용 통합계정**: 레거시에 AXOS 전용 계정+최소권한. 사람 계정 도용 금지 → 책임 추적성.

---

## 6. 첫 통합 권장 순서 (위험 낮은 것부터)

1. **Mail + File 읽기** → 문서 기억. (가장 안전, 즉시 체감: "AI가 우리 문서를 안다")
2. **ERP/SCM 재고·판매 읽기(배치)** → SCM/Sales Agent 인사이트. (쓰기 없음)
3. **알림형 쓰기** → 임계 초과 메신저/메일. (외부 영향 최소)
4. **승인형 쓰기** → 발주 추천→승인→ERP. (첫 "실행형" 가치 입증)
5. **조건부 자동** → 검증된 시나리오만 한도 내 자동.

이 순서가 docs/03의 AX전환 플레이북과 1:1로 맞물린다.
